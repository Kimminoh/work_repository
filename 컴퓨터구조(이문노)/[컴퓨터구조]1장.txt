컴퓨터의 5대 장치
- 중앙처리장치
- 기억장치
- 제어장치
- 입출력 프로세서(입출력 컨트롤러)
- 

명령어 set = ADD, SUB,...와 같은 명령어들(어셈블리언어)

고급언어를 명령어 셋으로 바꾸어주는 것 = 컴파일러

명령어의 집합 -> 실행코드(.exe) -> 실행코드를 메모리(DRAM)에 저장 -> 제어장치가 메모리에 있는 실행코드를 하나씩 읽어온다.(해석) -> 만약 ADD를 읽으면 ADD에 대한 하드웨어를 구현하여 제어신호 동작 -> 

* CPU chip 내 중앙처리장치가 들어간 것을 해당 중앙처리장치를 마이크로프로세서라고 한다.
또한 CPU chip 내에는 core가 또 여러개 나뉘어있다. core는 프로그램을 실행하기 위한 핵심적인 부분을 나누어놓은 것
또한 코어 개당 중앙처리 장치를 1개씩 가지고 있다! 그래서 코어의 갯수를 보고 멀티코어, 쿼드코어, 옥타코어 등으로 칭한다.  


[* 컴퓨터 구조적 설계(개념) *]

컴퓨터 조직 : 사용자가 이용할 수 없는 구조의 구현 (사람: 세포조직 마냥 조직하는 것!) 

컴퓨터 설계 : 컴퓨터 사양에 따라 적절한 하드웨어를 선택하고 그들간의 연겨ㄹ방식을 결정하여 컴퓨터하드웨어 설계
(기초작업! 호환성!)

컴퓨터 구조 : 사용자의 입장에서 구조나 동작에 중점! -> 사용자가 직접 이용할 수 있는 부분 (명령어 세트, 레지스터 , 메모리관리... 등등)
